# /skill-improver — スキル品質レビュー＆改善

`.claude/` 配下のスキル・コマンド・エージェントをAnthropicの公式ガイドラインに基づいてレビューし、改善する。

## 事前準備

レビュー前に `.claude/skills/skill-improver/references/guide.md` を読み込み、評価基準を把握する。

## 実行内容

### 1. 対象の特定

ユーザーが対象を指定していない場合、`.claude/` 配下のスキル・コマンド・エージェントを一覧表示し、選んでもらう。

```
スキル:      .claude/skills/*/SKILL.md
コマンド:    .claude/commands/*.md
エージェント: .claude/agents/*.md
```

「全部」「一括」と言われた場合は全ファイルを順にレビューする。

### 2. レビュー（7観点チェック）

`references/guide.md` の知識に基づき、以下の観点でレビューする。

| # | 観点 | 確認内容 |
|---|---|---|
| 1 | **description品質** | 「何をするか」+「いつ使うか（トリガー）」の両方があるか。具体的か |
| 2 | **トリガー網羅性** | ユーザーの自然な言い回し（日本語・英語）がカバーされているか |
| 3 | **ステップの具体性** | 各ステップが実行可能か。曖昧な指示（「適切に処理する」等）がないか |
| 4 | **エラーハンドリング** | よくあるエラーと対処法が網羅されているか |
| 5 | **Progressive Disclosure** | 本文が肥大化していないか。分離すべき詳細がないか |
| 6 | **確認フロー** | ユーザー承認が必要な箇所（外部投稿・破壊的操作）に確認ステップがあるか |
| 7 | **一貫性** | 他のスキル・コマンドとフォーマットや命名規則が統一されているか |

### 3. レビュー結果の提示

```
## レビュー結果: {スキル名}

| # | 観点 | 判定 | コメント |
|---|---|---|---|
| 1 | description品質 | OK / 要改善 | {具体的なコメント} |
| 2 | トリガー網羅性 | OK / 要改善 | ... |
| ... | ... | ... | ... |

### 改善提案
1. {具体的な改善案}
2. {具体的な改善案}
```

一括レビューの場合はサマリテーブルも出す:

```
| ファイル | 総合 | 要改善の観点 |
|---|---|---|
| wrap-up | A | — |
| morning | B | トリガー網羅性、エラーハンドリング |

総合判定: A = 問題なし / B = 軽微な改善あり / C = 要改善
```

### 4. 改善の適用

ユーザーの承認を得てから修正を適用する。
- 承認 → ファイルを編集
- 部分承認 → 指定された項目だけ適用
- キャンセル → 何もしない

## 注意

- スキル改善後は実際に動作テストすることを推奨する（「テストしてみましょうか？」と提案）
- Progressive Disclosureの原則に従い、SKILL.md本文は簡潔に保つ。詳細はreferences/に分離する
- descriptionの変更はトリガー動作に直接影響するため、慎重に行う
